Analisis Perbandingan: Klasifikasi pada Data Iris Sebelum dan Sesudah Diskritisasi K-Means

Catatan ini merangkum percobaan yang dilakukan untuk menganalisis dampak diskritisasi terhadap performa model klasifikasi. Proses diskritisasi dilakukan menggunakan metode unsupervised K-Means Clustering. Performa model dievaluasi pada dataset Iris sebelum (data kontinu) dan sesudah (data kategorikal) proses diskritisasi.

Dua algoritma klasifikasi yang digunakan untuk perbandingan adalah Naive Bayes dan Decision Tree.

1. Latar Belakang dan Tujuan

Data kontinu seringkali memiliki noise dan kompleksitas yang tinggi. Diskritisasi adalah proses mengubah fitur kontinu menjadi fitur diskrit atau kategorikal dengan membuat sejumlah interval (bins).

Tujuan dari eksperimen ini adalah:

  • Menerapkan metode K-Means Clustering sebagai teknik diskritisasi untuk mengubah fitur-fitur kontinu pada dataset Iris menjadi kategorikal.
  • Membangun model klasifikasi menggunakan Naive Bayes dan Decision Tree pada data asli (kontinu).
  • Membangun model klasifikasi yang sama pada data yang telah didiskritisasi (kategorikal).
  • Menganalisis dan membandingkan akurasi dari keempat model tersebut untuk memahami pengaruh diskritisasi terhadap performa masing-masing algoritma.

2. Proses Diskritisasi Menggunakan K-Means Clustering

K-Means adalah algoritma unsupervised learning yang bertujuan untuk mengelompokkan data ke dalam K cluster, di mana setiap data milik cluster dengan mean (centroid) terdekat. Dalam konteks ini, K-Means tidak digunakan untuk menemukan kelompok spesies, melainkan untuk membagi rentang nilai dari setiap fitur menjadi beberapa interval.

Proses ini dilakukan secara independen untuk setiap fitur (misalnya, sepal length (cm)):

  1. Inisialisasi: Untuk sebuah fitur, kita tentukan jumlah cluster K=3. Ini berarti kita ingin mengubah nilai-nilai kontinu pada fitur tersebut menjadi 3 kategori, yang dapat diinterpretasikan sebagai 'Rendah', 'Sedang', dan 'Tinggi'.

  2. Clustering: Algoritma K-Means dijalankan pada data satu dimensi dari fitur tersebut. Algoritma akan menemukan 3 centroid (pusat cluster) yang optimal dan memberikan label cluster (misalnya, 0, 1, atau 2) untuk setiap titik data.

  3. Pengurutan Label (Langkah Kunci): Label yang dihasilkan oleh K-Means (misalnya 0, 1, 2) awalnya bersifat arbitrer dan tidak memiliki urutan. Untuk membuatnya bermakna secara ordinal, kita perlu mengurutkannya. Caranya adalah dengan mengurutkan nilai centroid.

  • Cluster dengan nilai centroid terendah akan diberi label baru 0 (Rendah).
  • Cluster dengan nilai centroid tengah akan diberi label baru 1 (Sedang).
  • Cluster dengan nilai centroid tertinggi akan diberi label baru 2 (Tinggi).
  1. Transformasi: Nilai-nilai kontinu asli pada fitur tersebut digantikan dengan label ordinal yang baru (0, 1, atau 2). Proses ini diulangi untuk semua empat fitur, menghasilkan dataset baru di mana semua fiturnya bersifat kategorikal.

3. Desain Eksperimen Klasifikasi

Setelah data siap, kita melatih dan menguji empat model berbeda untuk perbandingan:

Model pada Data Asli (Kontinu)

  1. Gaussian Naive Bayes: Model ini dipilih karena asumsinya cocok untuk data kontinu, yaitu setiap fitur diasumsikan terdistribusi secara normal (Gaussian).
  2. Decision Tree: Algoritma ini secara inheren dapat menangani data kontinu dengan mencari titik potong (split point) optimal untuk memisahkan data.

Model pada Data Hasil Diskritisasi (Kategorikal)

  1. Categorical Naive Bayes: Setelah diskritisasi, fitur tidak lagi kontinu. Oleh karena itu, CategoricalNB digunakan karena model ini dirancang untuk bekerja dengan fitur-fitur yang bersifat diskrit atau kategorikal.
  2. Decision Tree: Algoritma ini juga sangat efektif pada data kategorikal. Ia akan membuat cabang berdasarkan kategori-kategori yang ada (misalnya, 'Rendah', 'Sedang', 'Tinggi') daripada mencari titik potong numerik.

Seluruh model dievaluasi menggunakan metrik akurasi pada data uji yang sama untuk memastikan perbandingan yang adil (fair comparison).

4. Hasil dan Kesimpulan Analisis

Eksperimen ini memberikan wawasan tentang bagaimana representasi data memengaruhi performa model.

Analisis Performa

  • Naive Bayes: Performa Naive Bayes seringkali sensitif terhadap asumsi distribusi data. Dalam kasus ini, dengan mengubah data menjadi kategorikal (CategoricalNB), kita melihat adanya perubahan akurasi. Tergantung pada seberapa baik data yang didiskritisasi memenuhi asumsi independensi kondisional, akurasinya bisa meningkat atau menurun.
  • Decision Tree: Pohon keputusan umumnya sangat kuat dan tidak memerlukan banyak pra-pemrosesan data. Namun, diskritisasi dapat menyederhanakan proses pembelahan (splitting) dan terkadang mengurangi dampak noise atau outlier, yang bisa sedikit mengubah akurasi. Pada dataset Iris yang sudah cukup "bersih", perubahannya mungkin tidak signifikan.

Analisis Kepentingan Fitur (Feature Importance)

  • Analisis pada Decision Tree menunjukkan bahwa fitur-fitur yang paling berpengaruh (umumnya petal length dan petal width) tetap menjadi fitur yang dominan bahkan setelah proses diskritisasi. Ini menandakan bahwa proses K-Means berhasil mempertahankan informasi penting yang terkandung dalam data.

Kesimpulan Akhir

Diskritisasi menggunakan K-Means adalah teknik yang efektif untuk mengubah data kontinu menjadi kategorikal dengan cara yang berbasis data (data-driven), bukan dengan interval yang sewenang-wenang.

Hasil dari eksperimen ini menunjukkan bahwa tidak ada jaminan bahwa diskritisasi akan selalu meningkatkan performa model. Dampaknya sangat bergantung pada:

  1. Algoritma yang Digunakan: Beberapa algoritma (seperti Naive Bayes) mungkin mendapat manfaat lebih besar dari data kategorikal.
  2. Karakteristik Dataset: Pada dataset yang polanya sudah jelas seperti Iris, kehilangan informasi akibat diskritisasi bisa jadi lebih merugikan daripada manfaat penyederhanaannya.

Pada akhirnya, keputusan untuk menggunakan diskritisasi harus didasarkan pada eksperimen dan pemahaman tentang bagaimana data dan model akan berinteraksi.

def discretize_with_kmeans(data: pd.Series, n_clusters: int, feature_name: str) -> tuple:
    """
    Melakukan diskritisasi pada sebuah fitur kontinu menggunakan K-means clustering.
    Label cluster diurutkan berdasarkan nilai pusatnya untuk memastikan keterurutan
    (misal: 0=rendah, 1=sedang, 2=tinggi).

    Args:
        data (pd.Series): Series pandas yang berisi data kontinu.
        n_clusters (int): Jumlah cluster (bins) yang diinginkan.
        feature_name (str): Nama fitur untuk keperluan output.

    Returns:
        tuple: Berisi array hasil diskritisasi dan model K-means yang telah dilatih.
    """
    # Reshape data agar sesuai dengan input scikit-learn (-1, 1)
    data_reshaped = data.values.reshape(-1, 1)

    # Inisialisasi dan latih model K-means
    # n_init='auto' untuk menggunakan implementasi yang paling efisien
    kmeans = KMeans(n_clusters=n_clusters, random_state=42, n_init='auto')
    clusters = kmeans.fit_predict(data_reshaped)

    # Urutkan label cluster berdasarkan nilai pusat (centroid)
    # Ini penting agar label memiliki makna (misal, 0 < 1 < 2)
    centers = kmeans.cluster_centers_.flatten()
    sorted_centers_indices = np.argsort(centers)

    # Buat pemetaan dari label asli K-means ke label baru yang berurutan
    cluster_mapping = {old_label: new_label for new_label, old_label in enumerate(sorted_centers_indices)}
    discretized_data = np.array([cluster_mapping[cluster] for cluster in clusters])

    # Tampilkan informasi hasil clustering
    print(f"\nHasil Diskritisasi untuk Fitur: {feature_name}")
    print(f"Pusat Cluster (diurutkan): {np.sort(centers)}")

    # Tampilkan rentang nilai untuk setiap cluster baru
    for i in range(n_clusters):
        cluster_values = data[discretized_data == i]
        print(f"  - Cluster {i} ('{['Rendah', 'Sedang', 'Tinggi'][i]}'): "
              f"Rentang [{cluster_values.min():.2f}, {cluster_values.max():.2f}], "
              f"Jumlah data: {len(cluster_values)}")

    return discretized_data, kmeans

# Lakukan diskritisasi pada setiap fitur
